== For Operators

=== Quick Start

See the **Configuration** section in link:../README.adoc[README.adoc] for:
- How to generate an Ed25519 signing key
- Environment variable configuration options
- Basic security guidelines

**TL;DR:**
[source,bash]
----
# Generate key
openssl genpkey -algorithm ED25519 -out signing_key.pem

# Configure
export LUCID_API_SIGNING_KEY_FILE=/path/to/signing_key.pem

# Start API
lucid-api
----

=== Architecture

The signing system consists of three layers:

1. **`Signer` trait** (`api/src/auth/signing.rs`)
   - Generic interface for any signing algorithm
   - Not specific to sessions or tokens
   - Future-proof for JWT, webhooks, etc.

2. **`Ed25519Signer`** (`api/src/auth/signing.rs`)
   - Concrete implementation using Ed25519
   - Handles PEM key loading and PKCS#8 parsing
   - Thread-safe for use in async contexts

3. **`SessionSigner<S: Signer>`** (`api/src/auth/signing.rs`)
   - Session-specific wrapper
   - Enforces `{session_id}.{signature}` format
   - Base64 encoding for HTTP compatibility
